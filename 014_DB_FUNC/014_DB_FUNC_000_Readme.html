数据库系统自带函数

说明

Oracle 


SELECT  必须要  FROM 一个表
单独测试某个函数，不检索表的情况下，需要 FROM dual
DB2 


SELECT  必须要  FROM 一个表
单独测试某个函数，不检索表的情况下，需要 FROM SYSIBM.SYSDUMMY1
SQL Server 


SELECT  可以不 FROM 一个表
直接 SELECT 函数()
MySQL 


SELECT  可以不 FROM 一个表
直接 SELECT 函数()

注：MySQL 中，也有一个  dual 表
用于 下面这种情况

INSERT INTO  test_book1
  SELECT 5, 'TEST', 'ABC' FROM dual
  WHERE NOT EXISTS( SELECT 1 FROM test_book1 WHERE id = 5);

因为 如果有  SELECT  和  WHERE  ，但是没有 FROM 的话，SQL报错。
SQLite 


SELECT  可以不 FROM 一个表
直接 SELECT 函数()
PostgreSQL 


SELECT  可以不 FROM 一个表
直接 SELECT 函数()
