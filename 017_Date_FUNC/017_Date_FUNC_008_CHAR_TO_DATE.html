SQLite 


使用函数进行处理，格式固定

Function  Equivalent strftime()  
date(...)   strftime('%Y-%m-%d', ...)  
time(...)   strftime('%H:%M:%S', ...)  
datetime(...)   strftime('%Y-%m-%d %H:%M:%S', ...)  
julianday(...)   strftime('%J', ...)
PostgreSQL 



根据返回值的需要， 使用  to_date 或者  to_timestamp 函数。


Test=# SELECT TO_DATE('2010.10.01', 'YYYY.MM.DD');
  to_date
------------
 2010-10-01
(1 行记录)



Test=# SELECT to_timestamp('2010.10.01 21:30:50', 'YYYY.MM.DD HH24:MI:SS');
      to_timestamp
------------------------
 2010-10-01 21:30:50+08
(1 行记录)







注：
to_timestamp 函数也能接受一个 double precision 参数， 把它从 Unix 纪元转换成 timestamp with time zone。 
（Integer Unix 纪元隐含地转换成了 double precision。）

Test=# select to_timestamp(200120400);
      to_timestamp
------------------------
 1976-05-05 14:00:00+09
(1 行记录)


逆运算为：（那个 -9*60*60 的原因是因为上面的查询，结果为 +09时区的 ）
Test=# select  EXTRACT(EPOCH from TIMESTAMP '1976-05-05 14:00:00') -9*60*60;
 ?column?
-----------
 200120400
(1 行记录)
