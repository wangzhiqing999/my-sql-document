数据库会话的基本参数设置 SET

允许将显式值插入表的标识列中

Oracle
DB2
SQL Server 

SET IDENTITY_INSERT [ database_name . [ schema_name ] . ] table { ON | OFF }

参数
database_name 
指定的表所在的数据库的名称。
schema_name 
表所属的架构的名称。
table 
包含标识列的表的名称。

备注
任何时候，一个会话中只有一个表的 IDENTITY_INSERT 属性可以设置为 ON。如果某个表已将此属性设置为 ON，
则对另一个表发出 SET IDENTITY_INSERT ON 语句时，
SQL Server 2005 将返回一个错误信息，指出 SET IDENTITY_INSERT 已设置为 ON，并报告已将其属性设置为 ON 的表。

如果插入值大于表的当前标识值，则 SQL Server 自动将新插入值作为当前标识值使用。
SET IDENTITY_INSERT 的设置是在执行或运行时设置的，而不是在分析时设置的。
